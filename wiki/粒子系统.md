# 粒子系统

> [!abstract] 摘要
> 粒子系统是 Cocos Creator 游戏引擎特效表现的基础，用于模拟火、烟、水、云、雪、落叶等自然现象以及发光轨迹、速度线等抽象视觉效果。Cocos Creator 3.8 同时支持 2D 和 3D 粒子系统，3D 粒子系统采用模块化架构，支持 CPU 与 GPU 两种渲染器。

## 核心概念

### 基本结构

粒子系统的基本单元是**粒子**，每个粒子具有位置、大小、颜色、速度、加速度、生命周期等属性。每一帧中粒子系统执行以下步骤：

1. 产生新的粒子并初始化
2. 删除超过生命周期的粒子
3. 更新粒子的动态属性
4. 渲染所有有效的粒子

粒子系统由以下部分组成：

- **发射器**：创建粒子并初始化属性
- **影响器**（动画模块）：更新粒子的运行时属性
- **渲染器**：生成渲染所需数据并渲染粒子
- **粒子类**：存储粒子属性
- **粒子系统类**：管理上述模块

### 模块化架构

Cocos Creator 3D 粒子系统通过模块来组织功能，每个模块独立管理一类粒子行为：

| 模块 | 核心功能 |
|------|----------|
| **主模块（ParticleSystem）** | 管理粒子生成、播放、更新、销毁，控制整体运行参数（时长、容量、循环、延迟等） |
| **发射器模块（ShapeModule）** | 控制粒子发射形状、方向和位置，支持盒子、圆形、锥体、球体、半球体五种发射形状 |
| **渲染模块（Renderer）** | 控制粒子渲染方式（公告牌、拉伸公告牌、水平/垂直公告牌、网格）和渲染器选择（CPU/GPU） |

### 动画模块（AnimatorModule）

粒子发射后的状态更新由以下九个动画子模块控制：

| 子模块 | 功能说明 |
|--------|----------|
| **颜色模块（ColorOvertime）** | 控制粒子在生命周期内的颜色渐变 |
| **大小模块（SizeOvertime）** | 控制粒子在生命周期内的大小变化，支持 XYZ 三维分轴缩放 |
| **速度模块（VelocityOvertime）** | 控制粒子在生命周期内的速度变化 |
| **加速度模块（ForceOvertime）** | 对粒子施加持续加速度，模拟风等外力效果 |
| **旋转模块（RotationOvertime）** | 控制粒子的旋转角速度 |
| **噪声模块（NoiseModule）** | 通过多层噪声增强粒子运动的随机性和自然感（仅 CPU 渲染器生效） |
| **贴图动画模块（TextureAnimation）** | 将贴图纹理作为帧动画播放，支持网格模式划分帧 |
| **限速模块（LimitVelocityOvertime）** | 限制粒子最大速度，超出时进行阻尼插值（仅 CPU 渲染器生效） |
| **拖尾模块（TrailModule）** | 在粒子运动轨迹上生成拖尾效果（仅 CPU 渲染器生效） |

## 关键细节

### 主模块核心属性

- **Duration**：粒子系统运行总时间
- **Capacity**：最大粒子数量
- **Loop**：是否循环播放
- **PlayOnAwake**：加载后是否自动播放
- **Prewarm**：循环模式下以已播放完一轮的状态开始（预热）
- **SimulationSpace**：控制粒子坐标计算的坐标系（World/Local）
- **SimulationSpeed**：控制整个粒子系统的更新速度
- **StartLifetime / StartColor / StartSize / StartSpeed / StartRotation**：粒子初始属性，大部分支持曲线和渐变色编辑
- **RateOverTime / RateOverDistance**：每秒/每单位距离发射的粒子数
- **Bursts**：在指定时间点爆发式发射指定数量的粒子，支持重复触发
- **GravityModifier**：重力系数（仅 CPU 粒子）

### 发射器形状

发射器模块支持五种形状，每种有独立的 Gizmo 可视化：

| 形状 | 特有属性 | 发射方式 |
|------|----------|----------|
| **盒子（Box）** | BoxThickness | Volume / Shell / Edge |
| **圆形（Circle）** | Radius, RadiusThickness, Arc | 从圆周到圆心按比例发射 |
| **锥体（Cone）** | Radius, Angle, Length | Base / Shell / Volume |
| **球体（Sphere）** | Radius, RadiusThickness | Volume / Shell |
| **半球（Hemisphere）** | 与球体相同 | Volume / Shell |

通用属性包括 Position、Rotation、Scale、AlignToDirection、RandomDirectionAmount、SphericalDirectionAmount、RandomPositionAmount。

### CPU 渲染器 vs GPU 渲染器

| 对比维度 | CPU 渲染器（默认） | GPU 渲染器 |
|----------|---------------------|------------|
| **实现** | 对象池维护粒子，逐帧生成 VB/IB 数据 | CPU 端提交初始参数，GPU 端执行模块计算 |
| **性能** | CPU 压力较大 | 减少 CPU 计算压力 |
| **限制** | 无特殊限制 | 不支持拖尾模块、限速模块、噪声模块 |
| **材质 Effect** | `builtin-particle` | `builtin-particle-gpu` |
| **启用方式** | 默认 | 勾选 UseGPU |

### 渲染模式（RenderMode）

- **Billboard**：粒子始终面向摄像机（公告牌模式）
- **StretchedBillboard**：面向摄像机但根据速度和大小进行拉伸
- **HorizontalBillboard**：粒子始终与 X-Z 平面平行
- **VerticalBillboard**：粒子始终与 Y 轴平行但朝向摄像机
- **Mesh**：使用指定的网格资源渲染粒子

### 粒子剔除

- **DataCulling**（资源剔除）：剔除未使用的模块数据，减小资源占用。在编辑器中勾选后生效，不适合需要在运行时动态开启模块的场景。
- **RenderCulling**（渲染剔除，v3.4 新增）：基于包围盒判断粒子发射器是否在摄像机可见范围内，支持三种剔除模式：
  - **Pause**：暂停模拟，恢复可见后继续
  - **Pause and Catchup**：暂停模拟，恢复可见后跳跃到当前时间
  - **Always Simulate**：始终模拟但不渲染

### 自定义材质命名规范

若需对粒子系统使用自定义材质，材质名称有特殊要求：
- CPU 粒子：材质名称必须包含 `particle-cpu`，如 `custom-particle-cpu`
- GPU 粒子：材质名称必须包含 `particle-gpu`，如 `my-particle-gpu`

### 相关组件

- **Billboard 组件**：用于渲染一个始终面向摄像机的矩形，属性包括贴图、高度、宽度、旋转角度
- **Line 组件**：渲染 3D 场景中给定点连接成的有宽度的线段，始终面向摄像机，支持宽度曲线和颜色渐变

## 与其他系统的关系

- **[[图形渲染]]**：粒子系统依赖渲染管线，使用内置 `builtin-particle` / `builtin-particle-gpu` / `builtin-particle-trail` 着色器进行渲染
- **[[材质系统]]**：粒子材质需要匹配对应的 Effect，CPU 和 GPU 粒子对材质 Effect 有严格限制
- **[[动画系统]]**：粒子的属性变化（颜色渐变、大小曲线、速度曲线）通过曲线编辑器和渐变色编辑器实现，与动画系统共享编辑范式
- **[[场景与节点系统]]**：粒子系统作为组件挂载在节点上，发射器位置、旋转、缩放相对于所在节点
- **[[脚本系统]]**：通过 ParticleSystem API 可在运行时控制粒子的播放、暂停、停止，动态调整模块属性

## 注意事项

- GPU 渲染器不支持拖尾模块、限速模块和噪声模块，若需要使用这些功能请使用默认的 CPU 渲染器
- 重力修正系数（GravityModifier）和速度修正系数（SpeedModifier）仅 CPU 粒子生效
- 粒子数量较少时不建议开启 RenderCulling，剔除运算每帧执行会有额外开销
- 启用 DataCulling 后，未勾选的模块数据会被移除，运行时无法再动态开启这些模块
- 自定义材质的 Effect 必须匹配渲染器类型，否则粒子无法正常渲染
- v3.4 中 `EnableCulling` 更名为 `DataCulling`
- v3.6.0 新增了 instance 支持，粒子 shader 的 vertex layout 有变更，升级时需注意自定义 shader 的兼容性
- Line 组件和 Billboard 组件归属于粒子系统目录下，但它们是独立组件，不依赖 ParticleSystem
