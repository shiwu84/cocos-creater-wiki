# 动画剪辑

> [!abstract] 摘要
> 动画剪辑（Animation Clip）是动画数据的声明式资源——它以时间轴 + 关键帧的形式描述节点属性如何随时间变化。它解决的根本问题是：将"对象在某时刻的某个属性值"这类离散的编辑操作，录制为一段连续可复用的插值动画数据，使得同样的动画效果可以挂载到不同的节点上播放、在多个场景中复用，或通过状态机逻辑在运行时动态选择和混合。动画剪辑是引擎动画系统的核心数据单元。

## 核心概念

### 两种来源

动画剪辑有两种创建渠道：

1. **引擎内创建**（可在动画编辑器中查看和编辑）
   - 通过 **动画编辑器** 直接录制节点属性的关键帧变化（位置、旋转、缩放、组件属性等）
   - 通过脚本进行程序化编辑，动态添加/修改动画曲线和关键帧
2. **外部导入的骨骼动画**（只读，不可在动画编辑器中编辑）
   - 第三方 DCC 工具（Blender、Maya）制作的骨骼动画
   - 模型文件（FBX/glTF）附带的动画，导入模型时自动生成动画剪辑子资源

> [!warning] 外部骨骼动画的限制
> 外部导入的骨骼动画剪辑在动画编辑器中节点处于锁定状态，不支持直接在引擎中编辑关键帧数据。如需修改，必须回到源 DCC 工具中重新导出，或在模型导入的 Animation 标签页中做简单的帧范围裁剪。

### 关键属性

动画剪辑在编辑器中可直接设置的属性：

| 属性 | 说明 |
|:---|:---|
| **WrapMode** | 播放循环模式：Normal（正向单次）、Loop（循环正向）、PingPong（来回往复）、Reverse（反向单次）、LoopReverse（循环反向） |
| **Sample** | 动画数据的采样帧率（FPS），默认 60。决定时间轴上每秒划分为多少帧，影响关键帧时间精度 |
| **Speed** | 播放速度倍率，默认 1。小于 1 慢放，大于 1 快放 |
| **Duration** | 动画总时长。前面的数字为 Speed=1 时的原始时长，括号内数字为当前 Speed 下的实际播放时长 |

### 节点数据组织

动画剪辑通过节点的**相对路径**来索引动画数据，忽略了根节点本身。这意味着：
- 动画剪辑记录的是"从挂载节点出发，哪个子节点的哪个属性在什么时间是什么值"
- 同一个动画剪辑可以挂载到不同的节点上，只要该节点下有相同相对路径的子节点结构
- 当节点重命名、删除或移动位置后，动画剪辑中的节点路径会失效，对应轨道在编辑器中显示为黄色不可编辑状态

### 关键帧编辑流程

1. 在节点上添加 Animation 组件，挂载或创建动画剪辑
2. 进入动画编辑模式（`Ctrl/Cmd+E`）
3. 在属性列表中添加需要动画化的属性（如 `position`、`cc.Sprite.spriteFrame` 等）
4. 在不同时间点修改属性值，系统自动在时间轴对应位置创建关键帧
5. 可在**关键帧编辑视图**和**曲线编辑视图**之间切换：前者以离散菱形显示关键帧，后者以贝塞尔曲线显示属性插值

> [!tip] 关键帧数据操作
> - **迁移节点数据**：当节点重命名导致动画数据丢失时，可通过右键菜单将原有动画数据迁移到新节点上
> - **复制粘贴**：节点和属性的动画数据支持跨 v3.x 编辑器间的复制粘贴，但粘贴不自动创建动画属性，源和目标需要至少有一个同名属性
> - **清空节点数据 / 移除属性轨道**：快速批量删除指定节点或属性的所有关键帧

### 动画剪辑的复用

动画剪辑本身是纯数据资源，不包含播放状态。播放状态保存在 **动画状态（AnimationState）** 对象中。同一个动画剪辑可以同时在多个 Animation 组件上播放，每个实例有独立的播放进度、速度和混合权重。

## Marionette 动画系统

v3.4 起引入的 Marionette 动画系统将动画剪辑纳入**状态机驱动**的流程中：
- **动画图（Animation Graph）** 定义动画状态（每个状态可绑定一个动画剪辑）和状态间的过渡条件
- **动画控制器（Animation Controller）** 在运行时根据游戏逻辑参数（如速度、是否在地面）自动切换动画状态
- 支持动画分层（多个层同时播放不同部位的动画，如上身射击 + 下身奔跑）、动画遮罩、叠加动画等高级特性

Marionette 让动画剪辑从"手动调用播放 API"升级到"声明式状态机自动控制"，是骨骼动画系统的核心进化。

## 实践要点

- 引擎内创建的动画剪辑可以自由编辑，适合 UI 动画、简单的 2D 角色动画（如 Sprite 帧切换）
- 外部骨骼动画的裁剪在模型导入的 Animation 标签页中进行：添加新剪辑 → 设置 Start/End 帧 → 裁剪出子区间
- Sample 值设置越高，动画精度越高但数据量越大。UI 动画通常 30 FPS 就足够，角色动画推荐 60 FPS
- 如果父节点下存在多个同名子节点，动画编辑器只支持编辑第一个同名子节点；需要分别编辑时，为每个子节点单独挂载 Animation 组件和动画剪辑
- 动画剪辑资源本身很轻量，大量动画剪辑的管理是项目组织的重要部分——建议按角色/功能模块建立目录结构
