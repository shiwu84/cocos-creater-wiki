# 物理系统

> [!abstract] 摘要
> Cocos Creator 3.8 提供了完整的 2D 和 3D 物理系统。3D 物理使用 Bullet 引擎，2D 物理使用 Box2D 引擎。通过 RigidBody（刚体）、Collider（碰撞体）、Joint（关节）等组件实现碰撞检测和物理模拟。

## 物理引擎

| 维度 | 引擎 | 特性 |
|------|------|------|
| 3D 物理 | Bullet | 完整的 3D 物理模拟，支持刚体、碰撞体、约束、角色控制器 |
| 2D 物理 | Box2D | 轻量高效的 2D 物理，支持刚体、碰撞体、关节 |

### 启用物理引擎

物理引擎默认不启用，需要在项目设置中开启：

```typescript
// 3D 物理
PhysicsSystem.instance.enable = true;

// 2D 物理
PhysicsSystem2D.instance.enable = true;
```

## 核心组件

### 刚体（RigidBody）

刚体使节点具有物理属性：

| 属性 | 说明 |
|------|------|
| Type | Static（静态）、Dynamic（动态）、Kinematic（运动学） |
| Mass | 质量 |
| Linear Damping | 线性阻尼 |
| Angular Damping | 角阻尼 |
| Use Gravity | 是否受重力影响 |

### 碰撞体（Collider）

定义物理形状用于碰撞检测：

| 3D 碰撞体 | 说明 |
|-----------|------|
| BoxCollider | 盒形 |
| SphereCollider | 球形 |
| CapsuleCollider | 胶囊形 |
| CylinderCollider | 圆柱形 |
| ConeCollider | 圆锥形 |
| MeshCollider | 网格形（性能消耗高） |
| TerrainCollider | 地形 |
| SimplexCollider | 凸包简化 |

| 2D 碰撞体 | 说明 |
|-----------|------|
| BoxCollider2D | 矩形 |
| CircleCollider2D | 圆形 |
| PolygonCollider2D | 多边形 |

> [!important] 必需组合
> 碰撞体必须挂载在拥有 RigidBody 组件的节点（或其父节点）上才能参与物理模拟。

### 关节（Joint）

| 3D 关节 | 说明 |
|---------|------|
| HingeJoint | 铰链关节（门、摆锤） |
| FixedJoint | 固定关节 |
| PointToPointConstraint | 点对点约束 |

### 角色控制器（Character Controller）

v3.8 新增的组件，简化了角色移动的物理处理：

- 自动处理斜坡、台阶碰撞
- 不需要 RigidBody 即可使用
- 适合第一人称/第三人称角色控制

## 碰撞检测

### 碰撞事件

```typescript
onCollisionEnter(event: ICollisionEvent) {
    // 碰撞开始
}

onCollisionStay(event: ICollisionEvent) {
    // 碰撞持续
}

onCollisionExit(event: ICollisionEvent) {
    // 碰撞结束
}
```

### 触发器

将 Collider 的 `isTrigger` 设为 `true`，碰撞时不会产生物理响应，只触发事件：

```typescript
onTriggerEnter(event: ITriggerEvent) {
    // 进入触发器
}
```

## 射线检测

从空间中的一点向某个方向发射射线，检测碰撞：

```typescript
// 3D 射线检测
const ray = new cc.geometry.Ray();
ray.o.set(x, y, z);
ray.d.set(dx, dy, dz);
PhysicsSystem.instance.raycast(ray, callback, mask);

// 2D 射线检测  
PhysicsSystem2D.instance.raycast(p1, p2, callback, mask);
```

## 物理材质

定义碰撞体的物理表面属性：

- **摩擦力（Friction）**：表面滑动摩擦力系数
- **弹性（Restitution）**：碰撞后的反弹系数

## 注意事项

> [!warning] 性能
> MeshCollider 性能消耗较大，尽量使用基础碰撞体（Box、Sphere、Capsule）的组合代替。

> [!tip] 物理步长
> 物理系统使用固定时间步长（默认 1/60 秒）。渲染帧率不影响物理模拟精度。

> [!danger] 缩放注意事项
> 修改带有碰撞体的节点的 Scale 可能导致非预期的物理行为。尽量通过修改碰撞体属性来调整形状大小。
