# 预制资源

> [!abstract] 摘要
> 预制件（Prefab）是一种序列化的场景对象模板资源。它解决的核心问题是：如何将精心配置的节点组合（含组件、属性和子节点层级）固化为一个可复用的"蓝图"，使得场景中能批量生成保持一致性的实例，同时允许每个实例在继承模板数据的基础上进行独立定制。预制件是实现"一次配置、多处复用"和"模板变动批量同步"的关键机制。

## 核心概念

### 预制件资源的四要素

| 概念 | 说明 |
|:---|:---|
| 预制件资源 | **资源管理器** 中的 `.prefab` 序列化文件，存储节点树、组件及组件数据的模板 |
| 预制件实例 | 从 **资源管理器** 拖入 **层级管理器** 后生成的节点对象，根节点标记为亮绿色 |
| 预制件编辑模式 | 双击预制件资源进入的独立编辑环境，所有非嵌套节点显示为亮绿色 |
| 嵌套预制件 | 预制件中的某个子节点本身是另一个预制件的实例，形成多层嵌套 |

### 数据继承与覆盖

预制件实例的数据遵循**分层覆盖**模型：

- **默认同步**：实例的数据源来自预制件资源的反序列化，属性值与模板保持一致
- **独立修改**：对实例的任何属性修改存储在实例自身中，不影响模板和其他实例
- **新增标记**：在实例下新增的子节点（右下角 `+` 标志）和新增组件（名称后 `+` 标志）仅存在于该实例

### 关键操作

| 操作 | 说明 |
|:---|:---|
| **从资源还原** | 将当前实例的数据恢复到预制件模板的最新状态，名称、位置和旋转不会还原 |
| **更新到资源** | 将当前实例的所有数据变更写回预制件资源模板，影响所有实例 |
| **取消关联** | 将预制件实例转为普通节点，完全脱离与模板的绑定关系（v3.4+ 支持批量操作） |
| **定位资源** | 快速在 **资源管理器** 中高亮定位到该实例所关联的预制件资源 |

### 嵌套预制件

预制件支持嵌套——一个预制件资源的子节点可以是另一个预制件资源的实例。嵌套预制件之间保持独立的模板关联：修改内层预制件资源会同步更新所有使用它的外层预制件实例。这一机制使得复杂游戏对象（如带装备的角色、带射击逻辑的武器）可以分层组织。

> [!warning] 硬性限制
> - 不允许在预制件实例中删除从预制件资源创建的节点
> - 不允许在预制件实例中更改资源节点的层级关系（如拖到其他父节点下）
> - 禁止预制件嵌套自身（会导致无限递归）

## 实践要点

- v3.8.5 新增属性检查器中的预制件预览功能，选中预制件即可在面板中查看渲染效果，极大提升资源审查效率
- 修改预制件资源后，所有场景中的实例会自动同步更新（编辑器会触发刷新）
- 多人协作时，预制件和场景一样是二进制资源，应避免同时修改同一预制件，按功能模块拆分预制件可降低冲突风险
- 删除组件的操作会记录为一笔"删除数据"存储在实例中，可通过还原按钮恢复或通过更新按钮将删除同步到模板
