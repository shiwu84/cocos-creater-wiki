# Rust 生命周期

> [!abstract] 摘要
> 生命周期是 Rust 用来保证"引用永远不会比它指向的数据活得更久"的机制。每个引用都有一个生命周期——引用有效的代码范围。大多数情况下编译器可以自动推断（生命周期省略规则），但当输入输出引用的关系不明确时——如返回两个参数中较长者的函数——需要显式标注生命周期参数来描述"返回值的生命周期 = 两个输入生命周期中较短的那一个"。生命周期注解不改变引用的实际存活时间；它只是将引用间的关系信息提供给借用检查器。

## 根本问题：防止悬垂引用

```rust
fn dangling() -> &String {
    let s = String::from("hello");
    &s // 错误：s 在函数结束时被释放
}
```

借用检查器通过比较作用域来判断引用是否有效。这在简单场景足够，但跨函数时——如 `fn longest(x: &str, y: &str) -> &str`——编译器不知道返回值借用了谁（x 还是 y），因此不知道返回值应该在什么时候失效。

生命周期注解解决的就是这个问题：将引用之间的关系显式化。

## 生命周期注解语法

```rust
&'a T        // 带有生命周期 'a 的不可变引用
&'a mut T    // 带有生命周期 'a 的可变引用
fn foo<'a>(x: &'a str) -> &'a str  // 函数签名中的生命周期参数
```

### 关键领悟

> 生命周期注解**不改变**引用的存活时间——它只是将多个引用的生命周期关系告诉编译器。函数的实际调用者（而非函数定义者）提供具体的生命周期。

### `longest` 示例

```rust
fn longest<'a>(x: &'a str, y: &'a str) -> &'a str {
    if x.len() > y.len() { x } else { y }
}
```

签名的含义：对于某个生命周期 `'a`，两个输入参数都至少存活 `'a`，返回值也至少存活 `'a`。实际上，`'a` 会被编译器确定为两个输入生命周期中**较短**的那一个——因为返回值必须对两个输入都有效。

## 生命周期省略规则

编译器有三条规则自动推导生命周期，免去大部分显式标注：

1. **每个引用参数获得自己的生命周期**：`fn foo(x: &i32)` → `fn foo<'a>(x: &'a i32)`
2. **仅一个输入生命周期 → 输出也用它**：`fn foo(x: &i32) -> &i32` → 自动推断为返回 `&'a i32`
3. **方法的 `&self` 生命周期 → 输出生命周期**：`fn method(&self, other: &str) -> &str` → 返回值的生命周期等于 `self` 的

当三条规则都用完后仍有引用未被赋予生命周期，编译器报错要求显式标注。

## 特殊生命周期

### `'static`

整个程序运行期间都有效的引用——字符串字面值是最典型例子：

```rust
let s: &'static str = "I live forever";
```

**常见误解**：编译器提示"需要 `'static` 生命周期"时，通常不是因为真的需要 static，而是你创建的引用与所需生命周期不匹配。不应盲目将引用标注为 `'static`，而应修复引用的生命周期关系。

### 结构体中的生命周期

包含引用的结构体需要生命周期注解：

```rust
struct ImportantExcerpt<'a> {
    part: &'a str,  // 结构体实例不能比 part 字段中的引用活得更久
}
```

### 方法中的生命周期

```rust
impl<'a> ImportantExcerpt<'a> {
    fn announce_and_return_part(&self, announcement: &str) -> &str {
        // 返回值生命周期 = self 的生命周期（第三条省略规则）
        self.part
    }
}
```

## 三条核心规则总结

| 规则 | 描述 |
|------|------|
| 输入规则 | 每个引用参数获得独立的生命周期参数 |
| 单参规则 | 只有一个输入生命周期 → 所有输出都使用它 |
| 方法规则 | 有 `&self`/`&mut self` → 所有输出生命周期 = self 的生命周期 |

## 与所有权借用的关系

生命周期是借用系统的时间维度：所有权决定**谁负责释放**，借用决定**谁能临时使用**，生命周期决定**借用的有效期**。三者构成 Rust 内存安全的完整拼图——所有权是宏观账本，借用在单个操作层面标记使用权，生命周期确保使用权不会跨越数据的有效期。

## 注意事项

- **函数参数的生命周期越短越好**：更短的生命周期标注意味着函数能接受更多类型的参数
- **返回值的生命周期至少与某个参数相关联**：不能返回函数内部创建值的引用（那是悬垂引用）
- **生命周期省略规则足够覆盖约 80% 的场景**：大多数函数不需要显式标注
