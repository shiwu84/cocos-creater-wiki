# 模型资源

> [!abstract] 摘要
> 模型资源是 DCC（Digital Content Creation）工具产出的 3D 资产在引擎中的序列化形式。它解决的根本问题是：如何将第三方工具（Blender、Maya、3ds Max 等）制作的复杂 3D 场景——包含几何网格、材质绑定、骨骼蒙皮、关键帧动画——无损地导入到游戏引擎中，并自动拆分为引擎能独立管理和优化的子资源（Mesh、Material、Animation、Skeleton、Prefab），使得美术工作流与程序工作流可以无缝衔接。

## 核心概念

### 支持的格式

| 格式 | 说明 |
|:---|:---|
| **FBX（.fbx）** | Autodesk 的交换格式，支持 FBX 2020 及更早版本。功能最全面，但格式私有、解析复杂 |
| **glTF 2.0（.gltf / .glb）** | Khronos 的开放标准，被称为"3D 界的 JPEG"。基于 JSON + 二进制数据，体积小、解析快，推荐优先使用 |

### 导入后自动生成的子资源

导入一个带完整动画的模型文件后，**资源管理器** 中会生成以下子资源树：

| 子资源 | 文件后缀 | 内容 |
|:---|:---|:---|
| **Mesh** | `.mesh` | 顶点位置、法线、切线、UV 坐标、三角形索引等几何数据 |
| **Material** | `.material` | 模型各子网格绑定的材质，默认使用模型内部嵌入的材质数据 |
| **Texture** | `.texture` | 模型引用的贴图文件，自动转换为引擎纹理格式 |
| **Animation** | `.animation` | 模型中包含的骨骼动画剪辑，可进一步裁剪和编辑 |
| **Skeleton** | `.skeleton` | 骨骼层级结构信息，供 SkinnedMeshRenderer 和动画系统使用 |
| **Prefab** | `.prefab` | 自动组装好的场景节点预制件，包含模型渲染器、动画组件等，可直接拖入场景 |

### Mesh 文件属性

选中 `.mesh` 文件可在 **属性检查器** 中查看几何统计信息：
- **Vertices**：顶点数量
- **Triangles**：三角形面数
- **UV**：纹理坐标通道索引
- **MinPos / MaxPos**：模型的包围盒最小/最大坐标

### 模型导入设置

选中模型的源文件（`.fbx` / `.gltf` / `.glb`），**属性检查器** 分为四个标签页：

#### Model 标签

| 属性 | 说明 |
|:---|:---|
| **Normals** | 法线导入策略：Optional（仅导入已有的）、Exclude（不导入）、Required（缺失时重新计算）、Recalculate（强制重新计算） |
| **Tangents** | 切线导入策略，选项同 Normals。法线贴图需要正确的切线空间 |
| **Morph Normals** | 形变目标（Morph Target / Blend Shape）的法线导入 |
| **Disable mesh split** | v3.6 起默认不拆分模型。超限骨骼时通过纹理传递替代 uniform 传递 |
| **Allow Data Access** | 是否允许 CPU 端读写网格数据（静态网格）。关闭则在提交 GPU 后自动释放 CPU 端数据以节省内存 |

**网格优化**（基于 meshoptimizer）：
- **顶点缓存优化**：重排三角形顺序提升 GPU 顶点缓存命中率
- **过度绘制优化**：减少像素被重复着色的次数
- **顶点提取优化**：提升顶点数据获取效率

**网格简化**：通过 `Target Ratio`（目标顶点比）和 `Error Rate`（误差率）控制 LOD 减面，但可能丢失 UV 排布。**网格压缩**支持二进制编码、zlib 压缩和浮点量化，减小包体。

#### Animation 标签

展示模型内所有动画剪辑的时间范围。可通过 `+` 按钮添加新的剪辑裁剪（设置 Start/End 帧），或通过 `-` 按钮删除不需要的动画。

#### Material 标签

- **提取材质**：勾选后将模型内嵌材质拉取为独立的 `.material` 文件，便于脱离模型独立编辑
- **材质提取目录**：指定提取后材质的存放路径
- **使用顶点色**：是否启用模型的顶点颜色通道
- **混合模式下的深度写入**：当 Alpha 模式为 Blend 时是否写入深度缓冲

#### FBX 标签（仅 FBX 格式）

- **与 1.x 版本兼容**：兼容 Cocos Creator 3D 1.x 的导入方式（注意可能影响已使用资源）
- **动画烘焙速率**：设定动画关键帧采样率（24/25/30/60 FPS），影响动画数据的精度和体积
- **优先使用文件时间范围**：是否优先使用 FBX 文件中记录的精确动画起止时间

### LOD（多层次细节）

网格导入器会检查子节点是否以 `_lodN` 结尾来识别 LOD 层级。也可以手动勾选 LODS 选项并使用自动 LOD 功能，通过设置 `LOD1`、`LOD2` 的三角面比例来自动生成不同精度的网格层级。

## 实践要点

- glTF 格式推荐优先使用——它是开放标准，解析快，文件小，在 Web 和移动端生态中支持最好
- FBX 模型导入时应关注 Y-up vs Z-up 坐标系问题，部分 DCC 工具导出的 FBX 需要勾选版本兼容选项
- **动画烘焙速率**的选择需要在精度和包体之间取平衡：角色动画通常 30 FPS 足够，快速动作可提升到 60 FPS
- 网格优化选项适合顶点数较多（>5000）的模型，小模型开启优化可能得不偿失
- 材质提取后会自动绑定到模型节点的 `Materials` 属性上，此过程不可逆，建议提取前确认目录结构
