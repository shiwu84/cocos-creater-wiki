# 场景与节点系统

> [!abstract] 摘要
> 场景是游戏环境的基本组织单元，由节点树和挂载在节点上的组件构成。Node 负责管理空间变换和层级关系，Component 赋予节点各种功能（渲染、物理、动画等）。Camera 组件是 3D 场景的必需元素。

## 场景（Scene）

场景是 `.scene` 格式的资源文件，保存了完整的节点树结构和所有组件数据。

### 场景制作工作流

1. 在场景编辑器中创建和摆放节点
2. 为节点添加和配置组件
3. 通过层级管理器组织节点树结构
4. 通过属性检查器调整组件参数

参见：[场景制作工作流程](raw/concepts/scene/index.md)

### 场景管理

- **加载场景**：`director.loadScene('sceneName')`
- **预加载场景**：`director.preloadScene('sceneName')`
- **场景资源**：场景在 `raw/asset/scene.md` 中有详细说明

## 节点（Node）

节点是场景的基础构成单位，以树状结构组织，支持父子层级。

### 节点属性

| 属性 | 类型 | 说明 |
|------|------|------|
| Position | Vec3 | 相对父节点的位置偏移 |
| Rotation | Quat | 旋转（四元数表示） |
| Scale | Vec3 | 缩放比例 |
| Mobility | enum | 可移动性（Static / Stationary / Movable） |
| Layer | bitmask | 可见性层级（与 Camera 的 Visibility 匹配） |

### Mobility 属性

节点的可移动性影响光照行为：

- **Static（静态）**：用于不移动的物体。可使用光照贴图（Lightmap），不参与运行时直接光照计算
- **Stationary（固定）**：光源只烘焙间接光，运行时计算直接光
- **Movable（可移动）**：不参与光照烘焙，使用光照探针（Light Probe）接收间接光

### Layer 属性

Layer 控制节点的可见性。节点的 Layer 必须包含在 Camera 的 Visibility 属性中才能被渲染。引擎提供 20 个用户自定义 Layer（User Layer 0 ~ 19）。

## 组件（Component）

组件是赋予节点功能的核心机制。通过组合多个组件实现复杂行为。

### 组件类型

| 类别 | 示例组件 | 功能 |
|------|---------|------|
| 渲染 | `MeshRenderer`, `Sprite`, `Label` | 图形渲染 |
| 光照 | `DirectionalLight`, `SpotLight` | 场景照明 |
| 物理 | `RigidBody`, `Collider` | 物理模拟 |
| 动画 | `Animation`, `SkeletalAnimation` | 动画播放 |
| 音效 | `AudioSource` | 音频播放 |
| UI | `Button`, `Layout`, `ScrollView` | 界面交互 |
| 自定义 | 用户定义的脚本组件 | 游戏逻辑 |

> [!important] 渲染组件规则
> 一个节点只能有**一个**渲染组件（MeshRenderer、Sprite、Label、Graphics、Mask、RichText、UIStaticBatch）。这是引擎的硬性限制。

### 组件生命周期

```
onLoad → onEnable → start → update (循环) → lateUpdate (循环) → onDisable → onDestroy
```

详见 [[脚本系统#生命周期]]。

## 坐标系与变换

Cocos Creator 使用右手坐标系：
- **X 轴**：向右
- **Y 轴**：向上
- **Z 轴**：向前（指向屏幕外）

节点的 Position、Rotation、Scale 都是**相对于父节点**的局部变换。世界坐标通过遍历父节点链计算得出。

## 节点树的渲染顺序

- 兄弟节点按在层级管理器中的顺序从上到下渲染
- 子节点在父节点之后渲染
- 可通过节点的 `siblingIndex` 属性调整渲染顺序
- 2D 渲染还受 `priority` 和 `sortingLayer` 影响

## 注意事项

> [!warning] 不要在一个节点上同时挂载光源组件和 MeshRenderer
> 虽然技术上可行，但不建议（可能导致非预期的光照行为）。应该分别使用独立节点。

> [!tip] 动态创建节点
> 脚本中使用 `instantiate(prefab)` 或 `new Node()` 创建节点，使用 `node.destroy()` 销毁。详见 [[脚本系统#创建和销毁节点]]。
