# 进阶主题

> [!abstract] 摘要
> Cocos Creator 的进阶主题涵盖引擎定制、热更新、数据存储、多语言国际化、网络通信、原生桥接（JSB）等高级功能。这些功能面向已有基础开发的用户，帮助实现更深度的引擎掌控和更复杂的产品需求。

## 核心概念

进阶主题围绕着对 Cocos Creator 引擎的深度定制和运行时环境的扩展能力，主要包括以下方向：

### 引擎定制

详见 [引擎定制工作流程](raw/advanced-topics/engine-customization.md)：

- Cocos Creator 拥有**两套引擎内核**：C++ 内核（原生平台）和 TypeScript 内核（Web/小游戏平台）
- 引擎框架层用 TypeScript 统一两套内核的差异
- TypeScript 引擎定制：修改引擎源码后，点击 **开发者 -> 编译引擎** 即可生效
- 原生引擎定制：需配置 external 依赖，使用 CMake 编译模拟器
- 偏好设置的引擎管理器中切换自定义引擎路径
- 还支持 [CMake 学习与配置](raw/advanced-topics/cmake-learning.md)、[原生引擎二次开发](raw/advanced-topics/native-secondary-development.md)

### 热更新

详见 [资源热更新教程](raw/advanced-topics/hot-update.md) 和 [热更新管理器](raw/advanced-topics/hot-update-manager.md)：

- 基于 AssetsManager 模块实现资源热更新，**仅适用于原生发布版本**（Web 平台可直接通过服务器更新）
- 通过 Manifest 文件比较服务端和本地版本的文件差异，以文件为单位下载更新
- Version 文件用于快速检查主版本号，确认是否需要完整下载 Manifest
- 热更新流程：build 生成 Manifest → 部署远程资源到服务器 → 客户端检测并下载差异文件 → 重启应用加载新资源
- 设置搜索路径使引擎优先使用下载的远程资源文件

> [!important] 热更新与 MD5 Cache
> 构建时不要勾选 MD5 Cache，否则热更新会失效。

### 数据存储

详见 [存储和读取用户数据](raw/advanced-topics/data-storage.md)：

- 使用 `sys.localStorage` 接口，遵循 Web Storage API 标准
- Web 平台直接调用 Web Storage API，原生平台使用 sqlite 存储
- 复杂对象需通过 `JSON.stringify` 序列化后存储
- 支持数据加密：先加密再存入 `localStorage`（如使用 encryptjs）
- 提供 `setItem`、`getItem`、`removeItem`、`clear` 四个基本操作

### 多语言国际化

详见 [i18n 游戏多语言支持](raw/advanced-topics/i18n.md)：

- i18n 扩展插件支持 Label 和 Sprite 组件的多语言切换
- 通过配置语言模板文件（如 `zh.ts`、`en.ts`）定义各语言的文本 key-value
- Label 节点挂载 `LocalizedLabel` 组件，Sprite 节点挂载 `LocalizedSprite` 组件
- 运行时通过 `i18n.init(language)` 动态切换语言
- 从 v3.6 起引擎已内置更强大的 [本地化（L10N）](raw/editor/l10n/overview.md) 功能

### 网络通信

- **HTTP 请求**：详见 [HTTP 请求](raw/advanced-topics/http.md)，支持 GET/POST 等标准 HTTP 方法
- **WebSocket**：详见 [WebSocket 介绍](raw/advanced-topics/websocket-introduction.md)、[WebSocket 客户端](raw/advanced-topics/websocket.md) 和 [WebSocket 服务器](raw/advanced-topics/websocket-server.md) — 提供前后端统一用法的 WebSocket 封装，连接字节跳动、支付宝、微信小游戏平台

### 原生桥接（JSB）

Cocos Creator 通过 JSB（JavaScript Binding）机制实现 TypeScript 与原生 C++ 代码的互操作：

- **JSB 自动绑定**：通过配置脚本自动生成绑定代码
- **JSB 手动绑定**：手动编写绑定逻辑以处理复杂的原生接口
- **sebind**：v3.8 新增的基于 C++17 的声明式绑定方案，替代传统 JSB 手动绑定
- **Java 反射 / OC 反射**：分别用于 Android 和 iOS 平台的原生调用
- **原生插件**：将第三方 SDK 封装为插件供脚本调用

## 关键细节

### 引擎定制工作流

1. 从 Cocos 引擎仓库获取匹配版本的源码
2. 将源码放到合适目录，配置 `external` 依赖
3. 在偏好设置中切换自定义引擎路径
4. 修改引擎代码
5. TypeScript 引擎：**开发者 -> 编译引擎**
6. 原生引擎：**构建发布 -> 构建 -> 编译**

### 热更新完整流程

```
构建原生包 → 生成 Manifest → 发布初始版本
    ↓
修改资源/代码 → 重新构建 → 生成新版本 Manifest → 部署到远程服务器
    ↓
客户端启动 → 检查 Version 文件 → 比对版本号 → 下载差异 → 重启应用
```

### JSB 绑定方案对比

| 方案 | 适用场景 | 复杂度 |
|------|---------|--------|
| 自动绑定 | 类接口有规律的原生模块 | 低 |
| 手动绑定（传统） | 复杂的原生接口 | 高 |
| sebind（v3.8） | 推荐的新项目，C++17 声明式 | 中 |
| Java/OC 反射 | 调用 Android/iOS 平台 API | 低 |

## 注意事项

> [!warning] 引擎版本匹配
> 自定义引擎的版本必须与使用的 Cocos Creator 编辑器版本匹配，否则会导致编译失败或运行时崩溃。

> [!danger] 热更新不适用 Web 平台
> 资源热更新（AssetsManager）仅适用于原生平台。Web 平台通过服务器请求即可实现版本更新。

> [!tip] 数据加密不能保证绝对安全
> `localStorage` 存储的数据即使加密，在客户端也并非完全安全。如需确保游戏存档不被破解，请使用服务器端数据存储。

> [!warning] sebind 需要 C++17
> sebind 绑定方案需要编译器支持 C++17 标准，适用于 Cocos Creator 3.8+。
