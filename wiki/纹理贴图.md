# 纹理贴图

> [!abstract] 摘要
> 纹理贴图（Texture2D）是 GPU 着色器采样的像素数据资源。它解决的根本问题是：如何将二维图像数据高效地传递到 GPU 显存中，使得渲染管线在执行片元着色时，能够通过 UV 坐标快速查询到每个像素的颜色值，从而将色彩、法线、金属度等表面属性"贴"到几何模型或 UI 精灵上。纹理是资产管线中体积最大、对内存和带宽影响最敏感的资源类型，其过滤与寻址设置直接决定了画面的清晰度和边缘表现。

## 核心概念

### 从 ImageAsset 到 Texture2D

纹理贴图不是直接由图像文件创建的。导入的图像文件（PNG、JPEG 等）首先成为 **ImageAsset**（图像源资源），然后系统将其转换为可在 GPU 中采样的 **Texture2D** 子资源。在 **资源管理器** 中展开图像资源左侧三角图标，可以看到自动生成的 `texture` 和 `spriteFrame` 子资源。选中 Texture2D 子资源后，在 **属性检查器** 中可配置以下属性。

### 过滤模式（Filter Mode）

当 Texture2D 的原始分辨率与屏幕映射尺寸不一致时（例如远处物体上的贴图被缩小），需要过滤算法来决定如何从相邻纹素（texel）采样出最终颜色：

| 模式 | 说明 |
|:---|:---|
| **Nearest** | 邻近过滤：取距离采样点最近的那个纹素颜色。速度快但边缘锯齿明显，适合像素风格游戏 |
| **Bilinear** | 双线性过滤：取周围 2×2 纹素矩阵做线性插值平均。过渡平滑但缩小到很小时可能产生摩尔纹 |
| **Bilinear with mipmaps** | 开启 Mipmap 的双线性过滤：在相邻的两个 Mipmap 层级间进行双线性过滤 |
| **Trilinear with mipmaps** | 三线性过滤：在连续两个 Mipmap 层级各自做双线性过滤后，再在两层之间做一次线性插值，最平滑但开销最大 |

### Mipmap

Mipmap 是预先计算并存储的逐级缩小版本图像序列（每一级为上一级的一半尺寸）。勾选 `Generate Mipmaps` 或将 Filter Mode 选为带 mipmap 的选项后，引擎会自动生成 Mipmap 链。其价值在于：

- **提升远距离渲染性能**：远处物体只需要采样低分辨率的 Mipmap 层级，提高 GPU 纹理缓存命中率
- **减少锯齿和摩尔纹**：低层级 Mipmap 本身就是平滑过的，避免了高频细节在远处缩小时的混叠伪影
- **内存代价**：额外占用约原始纹理 1/3 的内存空间

运行时可通过 `texture.setMipRange(minLevel, maxLevel)` 动态限制 Mipmap 层级的可用范围，以平衡带宽和画质。

### 寻址模式（Wrap Mode）

当纹理坐标超出 `[0, 1]` 范围时（如在地面砖块上使用大于 1 的 UV 平铺），寻址模式决定如何采样：

| 模式 | 说明 |
|:---|:---|
| **Repeat** | 重复平铺：在 UV 范围外重复图像内容。WebGL1.0 中要求纹理宽高为二次幂（POT），否则自动降级为 Clamp-to-edge |
| **Clamp-to-edge** | 边缘钳位：超出部分取最近边缘的颜色值，产生拉伸效果。导入为 sprite-frame 时会自动设置为此模式以避免透明边缘的黑边 |
| **Mirrored-repeat** | 镜像重复：与 Repeat 类似但每次重复时镜像翻转，消除接缝感 |

### 各向异性过滤（Anisotropy）

当表面与相机视角倾斜时（如从斜角看地面），单一方向的线性过滤不够用。Anisotropy 属性设置沿观察方向额外采样纹素的最大比例，值越高倾斜表面的纹理越清晰，但 GPU 开销也越大。

> [!warning] 平台兼容性
> - WebGL1.0 平台不支持非二次幂（NPOT）纹理的 Repeat 寻址模式，运行时会自动降级为 Clamp-to-edge，导致 `tilingOffset` 等材质属性失效
> - `setMipRange` 方法对 WebGL 和 GLES2 后端无效
> - 移动端 GPU 的 Anisotropy 支持参差不齐，建议测试后决定是否启用

## 实践要点

- 纹理是内存大户：一张 2048×2048 的 RGBA 纹理占用约 16MB 显存，启用 Mipmap 后约 21MB。合理设置纹理分辨率和压缩格式
- UI 纹理建议关闭 Mipmap（Filter Mode 选 Nearest 或 Bilinear），因为 UI 通常不涉及远近缩放
- 精灵帧（SpriteFrame）在导入时自动将关联的纹理寻址模式设为 Clamp-to-edge，避免透明边缘出现黑边
- 材质中的纹理属性需要 Effect 中声明了 `USE TEXTURE` 宏才会出现在属性检查器中
