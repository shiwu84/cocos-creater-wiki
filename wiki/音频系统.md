# 音频系统

> [!abstract] 摘要
> Cocos Creator 的音频系统通过 AudioSource 组件统一管理音乐和音效的播放。支持 `.ogg`、`.mp3`、`.wav`、`.mp4`、`.m4a` 等常见格式，提供编辑器拖拽配置和脚本 API 两种控制方式，并支持 Web Audio / DOM Audio 双模式加载。

## 核心概念

### 音频资源（AudioClip）
Cocos Creator 支持导入大多数常见音频文件格式，拖拽到资源管理器即可生成 AudioClip 资源。引擎根据音频长短分为两类：
- **音乐**：较长的背景音乐
- **音效**：短促的打击音效、爆炸音效等

支持格式：`.ogg`、`.mp3`、`.wav`、`.mp4`、`.m4a`，其中 `.ogg` 文件小、音质好但 iOS 不支持；`.mp3` 兼容性最好；`.wav` 无损但体积大。

### AudioSource 组件
AudioSource 是控制音频播放的核心组件，挂在节点上使用。主要属性：

| 属性 | 说明 |
|------|------|
| Clip | 绑定的 AudioClip 资源 |
| Loop | 是否循环播放 |
| PlayOnAwake | 是否激活时自动播放 |
| Volume | 音量（0~1） |

Cocos Creator 3.x 移除了 v2.x 的 `audioEngine` API，统一使用 AudioSource 组件。

## 关键细节

### 编辑器播放
1. 在节点上添加 AudioSource 组件
2. 将音频资源拖拽到 Clip 属性框
3. 设置 Loop、PlayOnAwake、Volume 等属性

### 脚本播放
通过 `getComponent(AudioSource)` 获取组件，调用 `play()`、`pause()`、`stop()` 方法控制播放。

### 音效播放（playOneShot）
`playOneShot` 接口用于播放短音效，具有以下特点：
- 播放时间短
- 可同时播放多个
- **一次性操作**，播放后无法暂停/停止，也无法监听结束回调

### 监听音频事件
v3.3.0 起支持事件监听，包括 `AudioSource.EventType.STARTED` 和 `AudioSource.EventType.ENDED`。

### Web 平台限制
需遵守浏览器 [Audio Play Policy](https://www.chromium.org/audio-video/autoplay)，即使设置了 `PlayOnAwake`，也需要首次用户交互后才能播放。

### 加载模式
- **Web Audio**（默认）：以 audio buffer 形式缓存，兼容性好
- **DOM Audio**：生成标准 audio 元素播放，iOS 上可能无法调整音量

若需强制 DOM Audio 模式，可通过 `assetManager.loadRemote` 指定 `audioLoadMode: AudioClip.AudioType.DOM_AUDIO`。

### 音频管理器模式
推荐使用单例模式的 AudioMgr 统一管理音频播放：创建常驻节点（`director.addPersistRootNode`），挂载 AudioSource，通过 `playOneShot` 和 `play` 分别处理音效和音乐。

## 与其他系统的关系

- [[资源系统]]：AudioClip 作为资源类型，通过 Asset Manager 加载管理
- [[场景与节点系统]]：AudioSource 是节点组件，依附于节点树
- [[脚本系统]]：通过 TypeScript API 控制音频播放和事件监听

## 注意事项

- `playOneShot` 是一次性操作，无法暂停或监听结束事件
- Web 平台需用户首次交互后才允许音频自动播放
- iOS 不支持 `.ogg` 格式
- DOM Audio 模式下 iOS 的 Volume 属性不生效
- v3.x 已移除 `cc.audioEngine`，需使用 AudioSource 组件
- 场景切换时建议使用常驻节点（`addPersistRootNode`）保持音频管理器不被销毁
