# Spine 动画

> [!abstract] 摘要
> Spine 骨骼动画资源解决 2D 角色动画的根本问题——如何用少量数据创建流畅、可混合、可换装的角色动画。相比于传统的序列帧动画（每帧一张完整图片），骨骼动画将角色拆分为骨骼和插槽，通过关键帧驱动骨骼变换来产生动画。这使动画数据量减少数十倍，且支持动画混合（如边跑边射击）、换装（替换插槽纹理）和运行时动态修改（如拉弓瞄准方向）。Cocos Creator v3.0+ 集成 Spine v3.8 运行时，支持 JSON 和二进制两种数据格式。

## 核心概念

### Spine 文件结构与装配

Spine 导出的动画资源由三部分组成：

| 文件 | 格式 | 内容 |
|------|------|------|
| **骨骼数据** | `.json` 或 `.skel`（二进制） | 骨骼层级、插槽定义、动画曲线、皮肤配置 |
| **图集数据** | `.txt` 或 `.atlas` | 纹理区域坐标、纹理文件映射 |
| **图集纹理** | `.png` | 角色各部位的纹理图集 |

三者在导入后自动关联。拖拽骨骼数据文件（.json/.skel）到 Spine 组件的 `SkeletonData` 属性即可完成配置，引擎自动加载关联的图集数据和纹理。v3.8.5 起新增属性面板预览功能，选中 Spine 资源即可在检查器中预览动画效果，无需运行游戏。

### 版本兼容性

| Creator 版本 | Spine 版本 | 注意事项 |
|-------------|-----------|---------|
| v3.0+ | v3.8 | 原生平台不支持 v3.8.75 特定版本 |
| v2.2 | v3.7 | — |
| v2.0.8 ~ v2.1 | v3.6 | — |
| v2.0.7 及以下 | v2.5 | — |

使用二进制 `.skel` 格式可以获得比 JSON 更小的文件体积和更快的加载速度，但调试时不便于人工阅读。

### Spine 组件与动画控制

Spine 组件是骨骼动画的播放器。设置 `skeletonData` 后，调用 `setAnimation(trackIndex, name, loop)` 播放指定动画。Spine 支持多轨道同时播放（如轨道0播放身体动画、轨道1播放面部表情），以及动画混合（`setEmptyAnimation` 清空轨道并通过混合时间控制过渡效果）。这些能力使 Spine 特别适合需要复杂动画状态机的角色系统。

## 实践注意事项

- **二进制 vs JSON 格式**：正式发布使用 `.skel` 二进制格式（更小更快）；开发阶段使用 `.json`（便于排查问题）。两种格式可共存，通过资源路径区分。
- **远程加载 Spine**：支持从服务器动态加载，纹理通过 `assetManager.loadRemote` 加载，骨骼和图集数据通过 `assetManager.loadAny` 加载（文本格式指定 ext: `.txt`，二进制指定 ext: `.bin`）。远程加载时需要手动构造 `SkeletonData` 对象并组装各部分。
- **原生平台限制**：v3.8.75 特定版本在原生平台不支持，使用 Spine 编辑器导出时需注意版本选择。
- **图集纹理隔离**：Spine 的 png 纹理不应与引擎的自动图集系统混合打包，会导致 UV 坐标错乱。
