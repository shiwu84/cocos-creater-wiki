# 引擎架构

> [!abstract] 摘要
> Cocos Creator 3.x 采用 **实体-组件系统（ECS）** 架构，以组合而非继承的方式构建游戏对象。引擎由场景管理、渲染管线、物理引擎、动画系统等多个子系统组成，通过节点树和组件挂载实现灵活的层级化开发。

## ECS 架构核心

Cocos Creator 的 ECS 架构由三个核心概念构成：

### 节点（Node）— 实体载体

节点是场景的基本组成单位，以树状结构组织：

- 每个节点包含位移（Position）、旋转（Rotation）、缩放（Scale）变换属性
- 子节点相对于父节点进行空间变换
- 节点的更新顺序是自顶向下逐级传递的
- 节点上可以挂载多个组件

### 组件（Component）— 功能模块

组件赋予节点具体的行为和能力：

- 一个节点可以挂载**多个**组件（组合模式）
- 一个节点只能挂载**一个渲染组件**（MeshRenderer / Sprite / Label / Graphics / Mask / RichText / UIStaticBatch）
- 每个组件负责一个独立的功能维度
- 组件通过 `@ccclass` 装饰器定义，继承自 `Component` 基类

### 场景（Scene）— 环境容器

场景是游戏环境因素的抽象集合：

- 场景是 `.scene` 资源文件，保存节点树结构和组件数据
- 游戏运行时可以加载和切换场景
- 每个场景默认包含一个挂载 Camera 组件的节点

```
场景 (Scene)
 └── 根节点 (Root Node)
      ├── Camera 节点 (含 Camera 组件)
      ├── 灯光节点 (含 DirectionalLight 组件)
      ├── 玩家节点 (含 MeshRenderer + 自定义组件)
      └── UI 节点 (含 Canvas + Sprite/Label 组件)
```

> [!note] 与 Unity 的区别
> Cocos Creator 的 ECS 实现与 Unity 的 GameObject-Component 模型相似但不完全相同。Cocos Creator 中节点和组件的耦合更紧密，且脚本组件既包含逻辑也包含数据。参见 [Unity 开发者快速入门指南](raw/guide/unity/index.md)。

## 子系统架构

Cocos Creator 3.8 的主要子系统及其关系：

| 子系统 | 核心职责 | 关键类/接口 |
|--------|---------|------------|
| [[场景与节点系统]] | 场景管理、节点树、生命周期 | `Scene`, `Node`, `Component`, `director` |
| [[资源系统]] | 资源加载、管理、缓存 | `Asset`, `AssetManager`, `resources` |
| [[图形渲染]] | 渲染管线、材质、Shader、光照 | `RenderPipeline`, `Camera`, `Material` |
| [[脚本系统]] | TypeScript 运行时、装饰器、事件 | `Component`, `@ccclass`, `EventTarget` |
| [[物理系统]] | 2D/3D 碰撞检测、刚体模拟 | `RigidBody`, `Collider`, `PhysicsSystem` |
| [[动画系统]] | 关键帧动画、骨骼动画、程序化动画 | `Animation`, `AnimationClip`, `SkeletalAnimation` |
| [[UI 系统]] | 2D UI 渲染、布局、交互 | `Canvas`, `Sprite`, `Label`, `Button` |

## 更新循环

引擎每帧执行以下更新顺序：

1. **事件派发** — 输入事件、自定义事件
2. **物理更新** — 物理模拟步进（固定时间步长）
3. **组件更新** — `update(dt)` 每帧调用
4. **延迟更新** — `lateUpdate(dt)` 在所有 update 之后调用
5. **渲染** — 场景渲染

```mermaid
graph LR
    A[事件派发] --> B[物理更新]
    B --> C[组件 update]
    C --> D[组件 lateUpdate]
    D --> E[渲染管线]
```

## 注意事项

> [!warning] 渲染组件互斥
> 一个节点上只能有一个渲染组件。如需叠加多个渲染效果，应创建子节点分别挂载渲染组件。

> [!tip] Mobility 属性
> 节点的 Mobility（Static / Stationary / Movable）影响光照烘焙和行为，详见 [[场景与节点系统#Mobility 属性]]。
