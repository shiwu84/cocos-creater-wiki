# 缓动系统

> [!abstract] 摘要
> Cocos Creator 的缓动（Tween）系统用于创建动态动画，支持位置、旋转、缩放、颜色等属性的渐变过渡。提供链式 API、丰富的内置缓动函数、延迟/队列/并行/重复等行为控制，以及 v3.8.4 新增的自定义动作、翻转、暂停恢复、时间缩放等高级特性。

## 核心概念

### 构造缓动
通过工具函数 `tween(target)` 或 `new Tween<T>(target)` 创建缓动实例。`tween` 是引擎提供的工具方法，不是 `Tween` 类的成员。

### 核心接口分类

| 类别 | 接口 | 说明 |
|------|------|------|
| 属性渐变 | `to` | 绝对值渐变 |
| 属性渐变 | `by` | 相对值渐变 |
| 瞬时动作 | `set` | 直接设置目标属性 |
| 瞬时动作 | `delay` | 延迟指定时间 |
| 瞬时动作 | `call` | 调用回调函数 |
| 瞬时动作 | `target` | 设置缓动目标 |
| 流程控制 | `sequence` | 顺序执行多个缓动 |
| 流程控制 | `parallel` | 并行执行多个缓动 |
| 流程控制 | `then` | 插入缓动到队列 |
| 流程控制 | `union` | 合并动作为一个整体 |
| 重复控制 | `repeat` | 重复执行指定次数 |
| 重复控制 | `repeatForever` | 永久重复执行 |
| 节点操作 | `show` / `hide` | 显示/隐藏节点 |
| 节点操作 | `removeSelf` | 移出场景树 |
| 节点操作 | `destroySelf` | 移出并销毁节点 |

### 缓动函数（Easing）
引擎内置大量缓动函数，如 `linear`、`quadIn`、`cubicOut`、`elasticIn`、`bounceOut` 等，通过 `ITweenOption` 的 `easing` 参数指定。

## 关键细节

### 链式 API
大部分接口返回 `this` 或新 `Tween` 对象，支持链式调用：
```typescript
tween(this.node)
    .to(1, { position: new Vec3(0, 10, 0) })
    .by(1, { position: new Vec3(0, -10, 0) })
    .delay(1)
    .by(1, { scale: new Vec3(2, 2, 2) })
    .start()
```

### ITweenOption 配置
缓动选项接口 `ITweenOption` 提供：
- `easing`：缓动函数（内置字符串或自定义函数）
- `onStart`：动作启动回调
- `onUpdate(target, ratio)`：每帧更新回调
- `onComplete`：动作完成回调
- `progress(start, end, current, ratio)`：自定义插值进度

### v3.8.4 新增特性

| 特性 | 说明 |
|------|------|
| `id` | 为动作添加数字标识，配合 `reverse`、`union` 使用 |
| `reverse` | 翻转整个缓动或特定标识的动作 |
| `union(fromId)` | 从指定标识开始合并动作 |
| `pause` / `resume` | 手动暂停/恢复缓动 |
| `pauseAllByTarget` / `resumeAllByTarget` | 按目标对象批量暂停/恢复 |
| `timeScale` | 缩放时间因子（0.5=减速一半，2=加速一倍） |
| `duration`（getter） | 获取缓动总时长 |
| `update(duration, cb)` | 固定时长的自定义动作 |
| `updateUntil(cb)` | 不确定时长的自定义动作 |
| `start(time)` | 从指定时间点开始缓动 |
| 字符串缓动 | 支持整型/浮点数字符串的属性渐变 |
| 属性级 easing/progress | 对单个属性自定义缓动函数或插值 |
| 函数返回目标值 | `to` 的目标值支持函数返回 |
| Auto pause/resume | 缓动目标为 Node 时，自动根据 active 状态暂停恢复 |
| Auto destroy | 缓动目标为 Node 时，监听销毁事件自动销毁 |

### 静态方法

| 方法 | 说明 |
|------|------|
| `Tween.stopAll()` | 停止所有缓动 |
| `Tween.stopAllByTag(tag, target?)` | 按标签停止缓动 |
| `Tween.stopAllByTarget(target)` | 按目标对象停止缓动 |
| `Tween.pauseAllByTarget(target)` | 暂停目标的所有缓动 |
| `Tween.resumeAllByTarget(target)` | 恢复目标的所有缓动 |

### 自定义类型缓动
缓动支持任意对象类型，需提供 `progress`（插值）、`clone`（克隆）、`add`（加法）、`sub`（减法）等自定义方法，以及 `legacyProgress: false` 启用新回调签名。

### 销毁管理
- 大部分缓动在完成后自动销毁
- `repeatForever` 需要手动调用 `stop()` 销毁
- v3.8.4+ 当缓动目标为 Node 时，自动监听销毁事件

## 与其他系统的关系

- [[动画系统]]：缓动系统提供程序化动画能力，与 Animation 组件形成互补
- [[场景与节点系统]]：缓动目标通常是 Node，支持节点显示/隐藏/移除/销毁
- [[UI 系统]]：缓动常用于 UI 元素的过渡动画、数字滚动等效果
- [[脚本系统]]：通过 TypeScript 链式 API 编写缓动逻辑

## 注意事项

- 通过 `node.position.x = 1` 直接赋值不会触发 dirty 标记，应使用 `setPosition` 或 `position` setter
- `repeat` 不传第二个参数时只重复最后一个动作；传入 `embedTween` 可重复指定动作
- `delay` 位置不同效果完全不同：在动作前表示延迟后开始，在动作间表示中间间隔
- `playOneShot` 播放的音效无法监听结束事件（与缓动无直接关系，参见[[音频系统]]）
- v3.8.4 前的版本切换场景需手动停止缓动，v3.8.4+ 引擎自动处理
- `repeatForever` 会导致缓动无法自动销毁，需手动停止
