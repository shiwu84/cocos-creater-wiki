# 场景资源

> [!abstract] 摘要
> 场景（Scene）是 Cocos Creator 中游戏内容的组织容器和序列化单元。它解决的根本问题是：如何将运行时节点的层次结构、组件数据以及全局渲染配置持久化为一组可管理的资源文件，并在游戏的生命周期中高效地加载、切换和释放。场景资源既是编辑期的创作画布，也是运行时通过 `director.loadScene` 驱动游戏流程切换的核心资源类型。

## 核心概念

### 场景文件与节点树

场景文件（`.scene`）以资源形式存在于 **资源管理器** 中，保存了该场景内完整的节点树结构和所有组件数据。在 **层级管理器** 中，`scene` 节点是场景节点树的根节点——它不仅是所有节点的顶层容器，还承载着场景级别的全局渲染属性。

### 全局渲染属性

选中 `scene` 根节点后，**属性检查器** 中可设置整个场景生效的全局属性：

- **环境光**：场景的基础光照，无方向性，确保暗面不会完全漆黑
- **阴影**：控制实时光影的参数，影响所有阴影投射光源
- **全局雾**：根据距离线性/指数衰减像素颜色，用于营造大气感和隐藏远处 LOD 切换
- **天空盒**：定义场景远距离的立方体贴图，影响环境反射和间接光照

### 场景切换与资源自动释放

场景切换通过 `director.loadScene` 实现运行时动态加载。**场景资源自动释放策略** 是内存管理的核心机制：

- 当从当前场景切换到下一场景时，如果开启了自动释放，则当前场景直接或间接引用的所有资源（脚本动态加载的除外）会被自动释放
- 如果某些资源需要在场景间保持存活（如全局单例持有的 ScriptableObject、跨场景 UI），可通过 `Asset.addRef` 增加引用计数来锁定这些资源
- 关闭自动释放时，场景引用的所有资源默认不释放，适合频繁来回切换的场景

> [!warning] 避免多人同时编辑
> 场景文件是二进制序列化资源，无法通过 Git 自动合并。多人同时修改同一场景文件极易造成冲突和数据丢失，应该在团队中通过功能分区或场景拆分来规避。

## 实践要点

- 创建场景推荐通过 **资源管理器** 右键菜单 `创建 -> Scene`，有助于保持项目目录结构清晰
- 场景切换是异步操作，切换期间可以显示 loading 界面，避免黑屏
- 大型项目应将场景按功能模块拆分（登录场景、主菜单场景、关卡场景），而非把所有内容塞进一个场景
- 使用 RenderTexture + UI 可以将 3D 场景内容渲染到 2D 界面上，实现 3D 角色预览等效果
