# 字体资源

> [!abstract] 摘要
> 字体资源解决游戏中文字渲染的根本问题——如何在不同的屏幕分辨率、渲染平台和语言环境下，高效地显示清晰可读的文字。Cocos Creator 提供三种字体方案：系统字体（零资源开销，调用平台原生渲染）、动态字体（TTF 格式，支持任意缩放和丰富字形）、位图字体（fnt + png，性能最优但固定尺寸），分别对应灵活性、品质和性能的不同侧重点，开发者可根据项目需求独立或组合使用。

## 核心概念

### 三种字体方案对比

| 特性 | 系统字体 | 动态字体（TTF） | 位图字体（fnt + png） |
|------|---------|---------------|---------------------|
| 文件体积 | 零 | 几 MB（完整字库） | 几十 KB（仅包含使用的字） |
| 渲染性能 | 平台依赖 | 中等（需动态生成纹理） | 最高（预生成纹理） |
| 缩放能力 | 矢量缩放 | 矢量缩放 | 固定尺寸，缩放会模糊 |
| 字形一致性 | 平台差异大 | 一致 | 一致 |
| 使用方式 | 勾选 Use System Font | 拖入 TTF 文件 | 拖入 fnt + png，png 设为 sprite-frame |

选择策略：需要多语言动态文本且体积不敏感 → TTF；UI 密集型且性能敏感（如伤害数字、聊天框）→ 位图字体；零成本快速原型 → 系统字体。

### 导入与配置流程

- **系统字体**：无需导入任何资源。在 Label 组件中勾选 `Use System Font` 即可，运行时调用平台的原生字体渲染 API。
- **动态字体**：将 `.ttf` 文件拖入资源管理器。拖拽到 Label 组件的 `Font` 属性即可使用，引擎根据文字内容动态生成字形纹理。
- **位图字体**：需要 `.fnt`（字形索引）和 `.png`（字形纹理）两个文件同时拖入资源管理器。**导入后必须将 png 文件的资源类型手动改为 sprite-frame**，否则无法渲染。位图字体由 Glyph Designer、Hiero、BMFont 等工具生成。

### 快速创建 Label 节点

两种方式：从层级管理器创建节点 → 选择"创建渲染节点/Label（文字）"→ 拖入字体资源；或直接从资源管理器拖拽字体文件到层级管理器，自动创建 Label 节点并关联该字体。切换 Label 组件的 `Font` 属性不会影响组件的其他设置。

## 实践注意事项

- **位图字体 png 类型转换**：导入后忘记将 png 设为 sprite-frame 是最常见的错误，会导致文字无法显示。
- **文件存放**：位图字体的 fnt 和 png 建议存放在独立目录中，不要与其他资源混在一起。
- **性能选择**：对性能敏感的 UI（如伤害数字）位图字体可通过合批优化远超动态字体；需要多语言动态文本时动态字体更灵活。
- **字库裁剪**：TTF 完整字库体积可能达数 MB，如果只使用部分字符，建议使用字体裁剪工具减小体积。
- **系统字体风险**：不同平台的系统字体字形不同（iOS 苹方 vs Android 思源），可能导致 UI 布局偏移。对视觉一致性要求高的项目应使用 TTF 或位图字体。
