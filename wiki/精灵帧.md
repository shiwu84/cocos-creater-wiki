# 精灵帧

> [!abstract] 摘要
> SpriteFrame（精灵帧）是 2D 渲染体系的基础图形容器资源。它解决的根本问题是：一张原始纹理通常是完整的位图，但 UI 和 2D 渲染常常只需要其中某个矩形区域，或者需要设置九宫格缩放边距来适应不同的显示尺寸。SpriteFrame 在 Texture2D 之上包装了一层裁剪信息和九宫格参数，使得同一张纹理可以产生多个不同的渲染区域——这就是精灵图集（Atlas / Sprite Sheet）的技术基础。

## 核心概念

### 资源层级关系

导入一张图像文件（如 `icon.png`）后，系统生成三层资源结构：

```
icon (ImageAsset)          # 图像源资源，原始像素数据
├── icon/texture (Texture2D)  # 纹理子资源，GPU 可采样的纹理对象
└── icon/spriteFrame (SpriteFrame)  # 精灵帧子资源，裁剪 + 九宫格信息
```

SpriteFrame 默认持有同级的 Texture2D 引用，Sprite 组件通过引用 SpriteFrame 来渲染图像。运行时可通过 `SpriteFrame.createWithImage(imageAsset)` 从图像源资源创建新的 SpriteFrame。

### 裁剪模式（Trim Type）

| 模式 | 说明 |
|:---|:---|
| **Auto（自动裁剪）** | 默认模式。自动分析图像边缘的透明像素并将其裁剪掉，减少空白区域的渲染开销。`Trim Threshold` 控制透明度阈值（0~1），低于该值的像素被裁剪 |
| **Custom（自定义裁剪）** | 手动指定裁剪矩形框（Trim X, Y, Width, Height），精确控制显示区域 |
| **None（无裁剪）** | 使用原图的完整尺寸，不做任何裁剪 |

### 九宫格（9-Slice）

通过设置 `Border Top/Bottom/Left/Right` 四个边距值，将图像划分为 3×3=9 个区域。当 Sprite 组件开启九宫格模式并拉伸时：
- 四个角保持原始大小不变
- 上下两条边在水平方向拉伸
- 左右两条边在垂直方向拉伸
- 中心区域在水平和垂直两个方向同时拉伸

这使得 UI 控件（按钮、对话框边框、进度条背景）可以在任意尺寸下保持边角的清晰度和比例，是 UI 自适应布局的核心技术。

### Packable 与动态合图

`Packable` 属性控制该 SpriteFrame 是否参与引擎的**动态合图**（Dynamic Atlas）和**自动图集**（Auto Atlas）构建。开启后，引擎会在运行时将多个小 SpriteFrame 的纹理合并到一张大图中，以减少 Draw Call。

> [!warning] 需要关闭 Packable 的场景
> 如果自定义 Effect 中使用了纹理的 UV 坐标进行特殊计算，合图会修改原始 UV 坐标导致效果错误。此时必须将该 SpriteFrame 的 `Packable` 设为 `false`。

### RenderTexture 集成

SpriteFrame 可通过 RenderTexture 将 3D 相机渲染内容映射到 2D UI 上。这是实现"在 UI 界面中显示 3D 模型预览"、"小地图相机渲染"等混合渲染场景的核心机制。

## 实践要点

- 旋转（Rotated）和偏移（Offset X/Y）属性在 Texture Packer 打包的资源中为只读，用于还原打包前的子图信息
- 从服务器动态加载图片时，只能拿到 ImageAsset，需要调用 `SpriteFrame.createWithImage(imageAsset)` 来构造 SpriteFrame
- 一个 ImageAsset 可以有多个 SpriteFrame 子资源（通过 Sprite Editor 手动裁剪多张子图），这对图集（Atlas）的使用至关重要
- SpriteFrame 对象本身很轻量（只存储裁剪元数据），纹理数据仍在 Texture2D 中共享
