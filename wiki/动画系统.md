# 动画系统

> [!abstract] 摘要
> Cocos Creator 3.8 提供了多层次的动画系统：基于关键帧的 Animation 组件、骨骼动画（Spine/DragonBones）、以及 v3.8 新增的程序化动画 Marionette 系统（动画图 + 状态机）。支持在编辑器中可视化编辑和预览动画。

## 动画类型

| 类型 | 说明 | 适用场景 |
|------|------|---------|
| **关键帧动画** | 基于 Animation 组件 + AnimationClip 资源 | 属性动画（位移、旋转、缩放、颜色等） |
| **骨骼动画** | Spine / DragonBones 骨骼动画 | 角色动画 |
| **程序化动画** | Marionette 动画图 + 状态机（v3.8 新增） | 复杂的角色状态管理和动画混合 |
| **变形目标动画** | Morph Target 顶点变形 | 面部表情等顶点动画 |

## Animation 组件

Animation 组件挂载到节点上，播放 AnimationClip 资源：

- 支持多种属性轨道：Position、Rotation、Scale、Color、Material 属性等
- 支持多种插值方式：Step、Linear、Bezier、Constant 等
- 支持循环播放、乒乓播放
- 可通过脚本控制播放/暂停/停止

```typescript
const anim = this.getComponent(Animation);
anim.play('walk');                  // 播放指定剪辑
anim.crossFade('run', 0.3);        // 带过渡的切换
anim.pause();                       // 暂停
anim.resume();                      // 恢复
anim.stop();                        // 停止
```

### 动画事件

可以在动画剪辑的特定帧上注册事件回调：

```typescript
const clip = anim.getState('attack').clip;
clip.on('frame-event', (frame) => {
    console.log(`触发帧事件于第 ${frame} 帧`);
});
```

## Marionette 动画系统（v3.8 新增）

Marionette 提供了**可视化程序化动画**能力：

### 动画图（Animation Graph）

- 类似 Unity 的 Animator Controller
- 定义参数（Float、Bool、Int、Trigger）
- 通过参数控制状态转移

### 动画状态机（State Machine）

- 状态包含一个或多个 Motion（动画剪辑或 BlendTree）
- 状态之间的 Transitions 由条件触发
- 支持多层（Layer），实现动画叠加

```
Idle ──(Speed > 0)──> Walk ──(Speed > threshold)──> Run
  ^                     │                              │
  └───(Speed = 0)───────┘                              │
  ^───────────────────(Speed = 0)──────────────────────┘
```

### Blend Tree

混合树允许多个动画按参数混合：

- **1D Blend**：按单一参数混合（如速度从 Idle → Walk → Run）
- **2D Blend**：按两个参数混合（如速度和方向）

## 骨骼动画

### Spine

- 支持 Spine 骨骼动画运行时
- 骨骼挂载、换装、动画混合
- 使用 `sp.Skeleton` 组件

### DragonBones

- 免费开源的骨骼动画方案
- 使用 `dragonBones.ArmatureDisplay` 组件

## 脚本控制

动画节点可通过脚本实现复杂的动画逻辑：

```typescript
// Tween 简单动画（无需 AnimationClip）
tween(this.node)
    .to(1, { position: new Vec3(100, 0, 0) })
    .to(0.5, { scale: new Vec3(2, 2, 2) })
    .call(() => console.log('done'))
    .start();
```

参见 [[缓动系统]] 了解 Tween API 详情。

## 注意事项

> [!tip] AnimationClip 和 Animator 的区别
> `Animation` 组件直接播放 AnimationClip，适合简单动画。Marionette 的 `AnimationController` 适合需要状态管理和动画混合的复杂场景。

> [!warning] 动画事件注册时机
> 动画事件的回调需要在 `play()` 之前注册，否则首帧事件可能丢失。
