# 声明文件

> [!abstract] 摘要
> 声明文件（`.d.ts`）是 TypeScript 生态的基石——它为纯 JavaScript 代码"披上类型外衣"。解决的根本问题是：TypeScript 编译器需要知道 JS 库的类型信息才能提供智能提示和类型检查，但 JS 源码中没有类型标注。`.d.ts` 文件作为"代码的说明书"只包含类型声明，不包含实现——编译器读取它们用于类型检查但不会产生任何运行时输出。DefinitelyTyped（`@types/` 包）社区维护了数万个 JS 库的声明文件，这是 TS 生态能如此快速扩张的关键基础设施。

## 根本问题：为无类型代码添加类型

TypeScript 的类型检查需要每个导入的值都有已知类型。但现实世界中绝大多数 npm 包是用纯 JavaScript 写的——它们没有类型。

声明文件就是"类型翻译层"：将 JS 库的 API 结构翻译为 TypeScript 类型描述。它本质上是一个**类型的契约**——"我向你保证，`lodash.map` 接收一个数组和一个函数，返回一个新数组"。

## `.d.ts` 文件的结构

```typescript
// types.d.ts
declare function greet(name: string): string;
declare namespace MyLib {
    function makeGreeting(name: string): string;
    let numberOfGreetings: number;
}
```

- `declare` 关键字：声明某物存在但不需要/不应在此提供实现
- `.d.ts` 文件**绝不能**包含可执行代码——只有类型声明
- 所有 `declare` 内容默认是全局可见的（除非文件是模块——包含 `import`/`export`）

## 为不同库结构编写声明

### 全局库

```typescript
// global.d.ts
declare function myGlobalLib(option: string): void;
interface MyGlobalLibOptions {
    timeout?: number;
}
```

不需要 `import`/`export`——声明直接注入全局作用域。

### 模块库

```typescript
// module.d.ts
export function doSomething(): void;
export interface Options {
    debug: boolean;
}
```

文件包含 `export` 关键字，使之成为一个模块声明。

### 插件/扩充

```typescript
// augment.d.ts
import 'some-lib';
declare module 'some-lib' {
    interface SomeInterface {
        newMethod(): void;
    }
}
```

扩充已有模块的类型——添加新方法或覆盖不准确的现有声明。

## DefinitelyTyped 与 `@types/`

DefinitelyTyped 是 GitHub 上的社区维护仓库，包含数万个 JS 包的声明文件。通过 npm 安装：

```bash
npm install --save-dev @types/lodash
```

包名映射：`lodash` → `@types/lodash`，`react` → `@types/react`。

关键约定：
- `@types/` 包的版本号应与对应 JS 包的版本号协调（通常使用 `.d.ts` 中 `// TypeScript Version: x.x` 的语义化兼容范围）
- 如果 JS 包自带 `.d.ts`（在 `package.json` 的 `types` 字段中指向），优先使用自带的

## 常见陷阱

### 不要用 `namespace` 在模块中

```typescript
// 错误
export namespace Foo {
    export function bar() {}
}

// 正确
export function bar() {}
```

模块已经是命名空间，嵌套 `namespace` 是多余的（声明文件中有历史例外）。

### `any` 是下下策

如果无法确定类型，用 `unknown` 而非 `any`——前者强制调用方进行类型检查，后者会关闭类型检查。

### 重载顺序重要

```typescript
declare function fn(x: string): string;  // 更具体的优先
declare function fn(x: number): number;
declare function fn(x: unknown): unknown; // 最通用的在最后
```

## 注意事项

- **`.d.ts` 文件不是源码的一部分**：它不产生 JS 输出，只提供编译期信息
- **`declare module '*'` 的滥用**：`declare module '*.css'` 虽然方便，但它会让所有 CSS 导入无类型（应使用更精确的方案）
- **声明文件是库的 API 合约**：修改声明可能破坏下游用户的类型检查——发布前仔细验证
- **JavaScript 项目也能使用声明文件**：通过 JSDoc 注解 + `checkJs` 选项，纯 JS 项目同样受益于声明文件
