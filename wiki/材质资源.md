# 材质资源

> [!abstract] 摘要
> 材质资源（Material）是连接着色器数据与渲染管线的桥梁资源。它解决的根本问题是：如何将离散的纹理贴图、颜色数值、光照参数等渲染数据，按照特定的着色流程组织在一起，然后传递给 GPU 以控制每个像素的最终颜色输出。材质让美术和开发者可以在不修改着色器代码的情况下，通过可视化属性面板调整物体的表面外观——粗糙度、金属度、法线强度等。

## 核心概念

### Effect + Technique + Pass 三层结构

材质的渲染行为由三层体系决定：

- **Effect（着色器资源）**：定义光照模型和渲染算法。默认使用内置 PBR 着色器 `builtin-standard.effect`，支持基于物理的金属/粗糙度工作流。项目中的自定义 Effect 文件可在此下拉框中选择
- **Technique**：一个 Effect 可包含多个 Technique，每个 Technique 针对不同平台或性能档次（如高端 PC 用全效果版本、移动端用简化版本）。切换 Technique 后 Pass 列表同步更新
- **Pass**：每个 Technique 包含若干 Pass（多 Pass 可实现描边、模糊等后处理效果）。每个 Pass 有独立的属性定义（`USE TEXTURE`、`USE INSTANCING` 等宏开关），需要先勾选定义宏才能看到对应的属性

### 动态优化选项

| 选项 | 说明 |
|:---|:---|
| **USE INSTANCING** | GPU Instancing：当场景中存在大量相同模型的实例时，启用此选项可大幅减少 Draw Call。但过度使用会因额外常量缓冲区维护开销反而降低性能 |
| ~~USE BATCHING~~ | 动态 VB 合并式合批，v3.6.2 起已从面板移除，合批功能由 INSTANCING 选项统一替代 |

### 属性缓存与迁移

材质在编辑器中会维护一份用户数据缓存。当切换 Effect 或 Technique 时，缓存中已有相同名称的属性会自动迁移到新的渲染配置中，最大程度保留用户的材质参数设置。修改材质后，面板右上角会出现保存和重置按钮——保存后不可再重置，编辑过程全程支持 `Ctrl+Z` / `Ctrl+Shift+Z` 撤销重做。

## 使用场景

### 3D 模型

通过网格渲染器（MeshRenderer/SkinnedMeshRenderer）的 `Materials` 属性数组指定。一个模型可以有多个材质槽，对应不同的子网格（sub-mesh）。

### 模型材质提取

从外部导入的 FBX/glTF 模型内可能嵌有材质，这些内嵌材质默认保存在模型文件内部且部分只读。如需独立编辑，可在模型资源的 **材质** 分页中勾选 **提取材质** 并指定输出目录，提取后的 `.material` 文件会自动绑定到模型节点上的 `Materials` 属性。

### 2D UI 与粒子

- 2D 渲染组件（Sprite、Label 等）通过 `Custom Material` 属性指定自定义材质，留空则使用引擎内置标准材质
- 粒子系统通过渲染器模块的 `ParticleMaterial` 和 `TrailMaterial` 属性分别控制粒子和拖尾的材质

## 实践要点

- 尽量不要在运行时频繁创建/销毁材质资源，应通过修改已有材质的 uniform 属性来实现外观变化
- 移动端应选择适合的 Technique（简化光照、减少纹理采样），同一 Effect 的不同 Technique 可共享大部分材质参数设置
- 材质预览支持切换预览模型（球体、立方体、平面等），便于在不进入场景的情况下直观调整效果
