# 脚本系统

> [!abstract] 摘要
> Cocos Creator 使用 TypeScript 作为主要脚本语言，通过 `@ccclass` 装饰器将类注册为组件。脚本组件具有完整的生命周期回调（`onLoad` → `start` → `update` → `onDestroy`），通过事件系统和装饰器实现组件间通信和数据绑定。

## 编程语言

Cocos Creator 3.x 使用 **TypeScript** 作为主要开发语言：

- 完整的类型检查，提升代码质量和 IDE 智能提示
- 引擎 API 提供完整的 TypeScript 类型定义
- 支持 ES Module 和 CommonJS 模块规范
- 通过 `tsconfig.json` 自定义编译选项

## 脚本基础

### 创建组件

使用 `@ccclass` 装饰器将类注册为 Cocos Creator 组件：

```typescript
import { _decorator, Component, Node } from 'cc';
const { ccclass, property } = _decorator;

@ccclass('MyComponent')
export class MyComponent extends Component {
    
    @property
    speed: number = 100;
    
    @property(Node)
    targetNode: Node | null = null;
    
    onLoad() {
        // 组件初始化
    }
    
    start() {
        // 首次 update 前调用
    }
    
    update(dt: number) {
        // 每帧调用
    }
    
    onDestroy() {
        // 组件销毁前调用
    }
}
```

### 装饰器

| 装饰器 | 用途 |
|--------|------|
| `@ccclass` | 将类注册为组件 |
| `@property` | 声明编辑器可见的属性 |
| `@executeInEditMode` | 允许编辑器模式下执行 |
| `@requireComponent` | 要求节点必须挂载指定组件 |
| `@disallowMultiple` | 禁止同一节点挂载多个该组件 |

`@property` 参数参考：[属性参数参考](raw/scripting/reference/attributes.md)

## 生命周期

```
场景加载 → onLoad → onEnable → start → update (循环) → lateUpdate (循环) → onDisable → onDestroy → 场景卸载
```

| 回调 | 时机 | 典型用途 |
|------|------|---------|
| `onLoad()` | 组件首次激活时，所有组件的 onLoad 串行执行 | 初始化引用、注册事件 |
| `onEnable()` | 组件/节点被激活时（可多次调用） | 启用相关功能 |
| `start()` | 所有 onLoad 执行完毕后，首次 update 前 | 依赖其他组件的初始化 |
| `update(dt)` | 每帧执行 | 逐帧逻辑 |
| `lateUpdate(dt)` | 所有 update 之后执行 | 跟随摄像机、动画后处理 |
| `onDisable()` | 组件/节点被禁用时 | 暂停逻辑 |
| `onDestroy()` | 组件/节点销毁时 | 清理资源、取消事件监听 |

## 节点和组件访问

### 访问节点

```typescript
this.node                           // 当前节点
this.node.parent                    // 父节点
this.node.children                  // 子节点数组
this.node.getChildByName('name')    // 按名称查找子节点
this.node.getPosition()             // 获取位置
this.node.setPosition(x, y, z)      // 设置位置
```

### 访问组件

```typescript
this.getComponent(Sprite)           // 获取当前节点上的指定组件
this.node.getComponent(Sprite)      // 等效方式
this.node.getComponentInChildren(Animation) // 在子节点中查找
this.node.getComponentInParent(RigidBody)   // 在父节点中查找
this.node.getComponents(Component)  // 获取所有指定类型组件
```

## 事件系统

Cocos Creator 有三层事件机制：

| 事件类型 | 说明 | 常用 API |
|---------|------|---------|
| 节点事件 | 挂载在 Node 上 | `node.on()`, `node.emit()`, `node.dispatchEvent()` |
| 全局事件 | 全局 EventTarget | `director.on()`, `game.on()` |
| 输入事件 | 键盘、鼠标、触摸 | `input.on(Input.EventType.xxx)` |

详见：[事件系统文档](raw/engine/event/index.md)

## 资源加载

```typescript
// Editor拖拽引用 (首选方式)
@property(SpriteFrame)
mySprite: SpriteFrame | null = null;

// 动态加载
resources.load('textures/background', SpriteFrame, (err, asset) => {});

// Bundle 加载
assetManager.loadBundle('gameAssets', (err, bundle) => {
    bundle.load('prefabs/player', Prefab, (err, prefab) => {});
});
```

## 注意事项

> [!warning] 装饰器执行顺序
> `@property` 的值在构造函数之后、`onLoad` 之前被注入。在构造函数中访问 `@property` 属性会得到初始值，不是编辑器设置的值。

> [!tip] 避免在 update 中频繁查找组件
> `getComponent()` 有性能开销，应在 `onLoad` 或 `start` 中缓存引用到成员变量。

> [!danger] destroy() 后不要访问节点
> 节点 `destroy()` 后所有属性变为无效。可以在 `isValid` 检查后再访问。
